# КРИТИЧЕСКАЯ ПРОБЛЕМА: Кириллический символ "с" в командах PowerShell

## ПРОБЛЕМА
Кириллический символ "с" (U+0441) постоянно заменяет латинский "c" (U+0063) в командах PowerShell, что вызывает ошибки:

```
CommandNotFoundException: Имя "с.\scripts\start_fastapi.ps1" не распознано как имя командлета
```

## РЕШЕНИЕ - ЧЕРНЫЙ СПИСОК
- **ЗАПРЕЩЕНО** использовать кириллический символ "с" (U+0441) в командах
- **ОБЯЗАТЕЛЬНО** использовать только латинский символ "c" (U+0063)
- **ВСЕГДА** проверять команды на наличие кириллических символов перед выполнением

## ПРИМЕРЫ ПРАВИЛЬНОГО ИСПОЛЬЗОВАНИЯ
✅ Правильно: `.\scripts\start_fastapi.ps1`
❌ Неправильно: `с.\scripts\start_fastapi.ps1`

✅ Правильно: `cd ..`
❌ Неправильно: `сd ..`

✅ Правильно: `Invoke-RestMethod`
❌ Неправильно: `Invoke-RestMethod` (с кириллическим с)

## ДЕЙСТВИЯ ПРИ ОБНАРУЖЕНИИ
1. Немедленно переписать команду с латинскими символами
2. Проверить все символы в команде
3. Выполнить команду заново
